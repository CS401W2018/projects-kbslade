CSS: Cascading Style Sheets:

cascading: who gets the final say if there's a conflict

  • *HTML Attributes: gets the ultimate and final say
  • *In line CSS: overrides any other system. You can use this on any tag displayed on page (property name:value; = color:red;)
  • * embedded CSS: add a style tag inside of the head. You have to put in a selector to show what you are changing the style to (all text in paragraphs, etc)
  • external CSS: add a css page for the styles and add the link in the
    • * DON'T USE

Engagement Questions:

  1. Explain the difference between inline, embedded, and external CSS.

    2. Inline, embedded and external CSS is the chain of command that the html 'cascades' to. Inline has more say than embedded and external. Embedded CSS has more than inline CSS.

  2. Explain why you should only use external CSS.

    3. You should only use external CSS because it keeps all our pages on our site the same. It allows the stylists to change things up one time rather than for each page.

  3. What is a selector used for?

    4. The selector indicates which part of the text style be changed. For example, if the css page shows: "main p {...}", the "main p" indicates that the paragraphs in the main body of the html page will be affected by whatever is put within the curly parentheses{}.
